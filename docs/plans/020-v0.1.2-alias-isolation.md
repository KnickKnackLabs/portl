# v0.1.2 Alias Isolation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> superpowers:subagent-driven-development (recommended) or
> superpowers:executing-plans to implement this plan task-by-task.
> Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Swap the `alias_store` backing storage from
`aliases.sqlite` (rusqlite, bundled) to `aliases.json`
(`serde_json` + `fd-lock`), **fully remove `rusqlite` from the
workspace**, and ship as v0.1.2 so we can test the
`rusqlite`-as-macOS-release-crash-cause hypothesis in isolation.
Spec: [`../specs/160-v0.1.2-alias-isolation.md`](../specs/160-v0.1.2-alias-isolation.md).

**Architecture:** The `crate::alias_store` module stays; only
its implementation changes. Public API (`AliasStore`,
`AliasRecord`, `StoredSpec`, `now_unix_secs`, `default_db_path`)
is preserved **byte-for-byte** so call sites in `docker.rs`,
`peer.rs`, and `slicer.rs` need no edits. Persistence becomes a
single JSON file protected by `fd-lock`: writers take an
exclusive lock for the read-modify-write-rename cycle, readers
take a shared lock.

**No backwards compatibility.** Pre-v0.1.2 `aliases.sqlite`
files are ignored silently (no migration tooling, no log line).
This is a pre-1.0 project with no external users beyond the
maintainer; `aliases.sqlite` is recreatable with a handful of
`portl ticket accept --save-as` invocations. Keeping the
module API stable is enough.

**Tech Stack additions:** `fd-lock = "4"` at the workspace
level. `serde_json` is already present. No `chrono`, no
`tempfile` (atomic rename uses a sibling `.tmp` file via
`std::fs`). `rusqlite` and `libsqlite3-sys` are removed.

## Task 2: In-place swap of `alias_store` to JSON + `fd-lock`

**Files:**
- Rewrite: `crates/portl-cli/src/alias_store.rs` (module name
  stays, public API stays).

**Public API that MUST remain byte-identical to v0.1.1:**

| Item | Signature |
| --- | --- |
| `pub struct AliasRecord` | fields `name, adapter, container_id, endpoint_id, image, network, created_at: i64` (+ derives `Debug, Clone, PartialEq, Eq, Serialize, Deserialize`) |
| `pub struct StoredSpec` | fields `caps: Capabilities, ttl_secs: u64, to: Option<[u8; 32]>, labels: Vec<(String, String)>, root_ticket_id: Option<[u8; 16]>, ticket_file_path: Option<PathBuf>, group_name: Option<String>, base_url: Option<String>` |
| `pub struct AliasStore` | constructor `pub fn new(db_path: PathBuf)`; `impl Default` |
| `AliasStore::save(&self, &AliasRecord, &StoredSpec) -> Result<()>` | unchanged |
| `AliasStore::get(&self, &str) -> Result<Option<AliasRecord>>` | unchanged |
| `AliasStore::get_spec(&self, &str) -> Result<Option<StoredSpec>>` | unchanged |
| `AliasStore::list(&self) -> Result<Vec<AliasRecord>>` | returns records ordered by name (matches the v0.1 `ORDER BY name ASC`) |
| `AliasStore::remove(&self, &str) -> Result<()>` | unchanged |
| `pub fn default_db_path() -> PathBuf` | now returns `<portl_home>/aliases.json` |
| `pub fn now_unix_secs() -> Result<i64>` | unchanged |

Call sites in `commands/peer.rs`, `commands/docker.rs`, and
`commands/slicer.rs` therefore need **zero** edits.

**On-disk layout (new):**

```
$PORTL_HOME/
├── aliases.json       # the data
└── aliases.json.lock  # companion fd-lock target
```

**JSON schema:**

```json
{
  "version": 1,
  "aliases": {
    "<name>": {
      "record": {
        "name": "<string>",
        "adapter": "<string>",
        "container_id": "<string>",
        "endpoint_id": "<string>",
        "image": "<string>",
        "network": "<string>",
        "created_at": 1740000000
      },
      "spec": {
        "caps": { ... Capabilities ... },
        "ttl_secs": 60,
        "to": "<32-byte hex>" | null,
        "labels": [["k","v"], ...],
        "root_ticket_id": "<16-byte hex>" | null,
        "ticket_file_path": "<path>" | null,
        "group_name": "<string>" | null,
        "base_url": "<string>" | null
      }
    }
  }
}
```

`to` and `root_ticket_id` are stored as hex strings to keep the
file grep-friendly; round-trip is handled by custom `serde` or
by thin wrapper types that re-use `hex::encode` / `hex::decode`
(matches the v0.1 SQLite encoding).

- [ ] **Step 1: Replace the module implementation**

Delete the body of `crates/portl-cli/src/alias_store.rs` and
rewrite as follows (skeleton; fill imports to match):

```rust
use std::collections::BTreeMap;
use std::fs::{File, OpenOptions};
use std::io::{Read, Write};
use std::path::{Path, PathBuf};
use std::time::{SystemTime, UNIX_EPOCH};

use anyhow::{Context, Result};
use fd_lock::RwLock;
use portl_core::id::store;
use portl_core::ticket::schema::Capabilities;
use serde::{Deserialize, Serialize};

const DB_FILE: &str = "aliases.json";
const LOCK_FILE: &str = "aliases.json.lock";

// AliasRecord / StoredSpec definitions copied verbatim from v0.1.1
// (same fields, same derives). See the "Public API" table above.

#[derive(Serialize, Deserialize, Default)]
struct Root {
    version: u32,
    aliases: BTreeMap<String, Entry>,
}

#[derive(Serialize, Deserialize)]
struct Entry {
    record: AliasRecord,
    spec: StoredSpec,
}

// Hex serialization helpers for `to: Option<[u8; 32]>` and
// `root_ticket_id: Option<[u8; 16]>`. A pair of
// `serialize_with` / `deserialize_with` modules handles null ⇔
// None and hex-string ⇔ byte array.

pub struct AliasStore {
    path: PathBuf,
    lock_path: PathBuf,
}

impl Default for AliasStore {
    fn default() -> Self {
        Self::new(default_db_path())
    }
}

impl AliasStore {
    pub fn new(db_path: PathBuf) -> Self {
        let lock_path = db_path.with_file_name(LOCK_FILE);
        Self { path: db_path, lock_path }
    }

    pub fn save(&self, alias: &AliasRecord, spec: &StoredSpec) -> Result<()> {
        self.with_write_lock(|root| {
            root.version = 1;
            root.aliases.insert(
                alias.name.clone(),
                Entry { record: alias.clone(), spec: spec.clone() },
            );
            Ok(())
        })
    }

    pub fn get(&self, name: &str) -> Result<Option<AliasRecord>> {
        Ok(self.read()?.aliases.remove(name).map(|e| e.record))
    }

    pub fn get_spec(&self, name: &str) -> Result<Option<StoredSpec>> {
        Ok(self.read()?.aliases.remove(name).map(|e| e.spec))
    }

    pub fn list(&self) -> Result<Vec<AliasRecord>> {
        Ok(self.read()?.aliases.into_values().map(|e| e.record).collect())
    }

    pub fn remove(&self, name: &str) -> Result<()> {
        self.with_write_lock(|root| {
            root.aliases.remove(name);
            Ok(())
        })
    }

    fn read(&self) -> Result<Root> {
        // If the data file is missing, return an empty Root.
        // A stray `aliases.sqlite` in the same directory is
        // silently ignored (we never open it).
        if !self.path.exists() {
            return Ok(Root::default());
        }
        // Shared fd-lock on the data file itself. If an exclusive
        // writer holds the lock, this blocks until the writer
        // finishes its rename — readers then see the new contents.
        let file = File::open(&self.path)
            .with_context(|| format!("open alias store {}", self.path.display()))?;
        let mut lock = RwLock::new(file);
        let guard = lock.read().context("acquire alias store read lock")?;
        let mut s = String::new();
        (&*guard).read_to_string(&mut s).context("read alias store")?;
        drop(guard);
        if s.trim().is_empty() {
            return Ok(Root::default());
        }
        serde_json::from_str(&s).context("parse alias store JSON")
    }

    fn with_write_lock<F>(&self, f: F) -> Result<()>
    where
        F: FnOnce(&mut Root) -> Result<()>,
    {
        if let Some(parent) = self.path.parent() {
            std::fs::create_dir_all(parent)?;
        }
        // Exclusive fd-lock on a companion file. Using a separate
        // lock file means fs::rename() can replace the data file
        // without the lock fd pointing at the replaced inode.
        let lock_file = OpenOptions::new()
            .read(true)
            .write(true)
            .create(true)
            .truncate(false)
            .open(&self.lock_path)?;
        let mut lock = RwLock::new(lock_file);
        let _guard = lock.write().context("acquire alias store write lock")?;

        let mut root = self.read()?;
        f(&mut root)?;

        let tmp = self.path.with_extension("json.tmp");
        let mut out = File::create(&tmp)?;
        serde_json::to_writer_pretty(&mut out, &root).context("encode alias store JSON")?;
        out.sync_all().context("fsync alias store tmp")?;
        std::fs::rename(&tmp, &self.path).context("rename alias store tmp into place")?;
        Ok(())
    }
}

pub fn default_db_path() -> PathBuf {
    store::default_path()
        .parent()
        .map_or_else(|| PathBuf::from(DB_FILE), |parent| parent.join(DB_FILE))
}

pub fn now_unix_secs() -> Result<i64> {
    Ok(i64::try_from(
        SystemTime::now()
            .duration_since(UNIX_EPOCH)
            .context("system clock is before unix epoch")?
            .as_secs(),
    )?)
}

// Hex helper modules: `hex_opt_32` and `hex_opt_16`.
// Each exports serialize / deserialize functions that map
// Option<[u8; N]> <-> Option<String> via hex::encode / hex::decode.
```

The two hex-helper modules can be a single generic `hex_opt<N>`
module; a straightforward `serialize_with` / `deserialize_with`
pair on each `StoredSpec` field is also fine. Whichever form has
fewer lines wins.
